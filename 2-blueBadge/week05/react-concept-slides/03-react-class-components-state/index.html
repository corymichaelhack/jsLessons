Class Components and State

Class Components

What are Class Components?

Class Components are our very own React components. They are created using JavaScript's class system and keyword, and are capable of tracking local information, and typically define logic functions to use locally, or pass off to child components.

What is 'local'?

When we use 'locally', we are referring to within a certain component, or locale. When it comes to class components, their locale is themselves. They can also pass their information and logic along to other components or locales as well.


class SignUp extends React.Component {
    constructor() {
        super();
        this.state = {
            title : 'Hello React'
        }
    }

    render() {
        return (
            <div>
            </div>
        )
    }
}

Extends & React Component


            class SignUp extends React.Component {

Whenever we create a new class component, we create it based off of the React Component that is provided to us via React itself. Creating our component with the React Component as it's blueprint has many advantages, namely the LifeCycle methods our components will inherit.

Life Cycle Methods

Life Cycle Methods, sometimes called Life Cycle Hooks, are methods defined by the parent React component. The compiler for our app will actually call these methods at different points in the component's life cycle. If we implement these methods, we can get different kinds of control over our components.

Constructors


            constructor() {
                super();
                this.state = {
                    title : 'Hello React'
                }
            }

Constructors are a part of classes in any programming language. The constructor, as it's name implies, has the responsibility of setting up the class, and constructing any values that should be present at the creation of the class.

Super


                            super();

When we inherit from another class, often times we need to make sure the properties and methods the parent defines are set. If we don't plan on changing how our custom class is set up, we can let JavaScript call the parent's constructor, and move on. If we want to change it however, we'll need to use super() to call the parent's constructor, before we start defining custom changes to our classes.

Render


                render() {
                    return (
                        <div>
                        </div>
                    )
                }

The render() method is the only method a stateful component has to implement. It's also a life cycle method! The render() method takes care of displaying our markup code to the page, just make sure you return it.

What is 'this'?

The keyword this is used to refer to contexts. this always refers to it's own context. So inside class components, this usually refers to the component itself.

Using Props


        <button onClick={this.props.onClick}>Click me!</button>

The practice of using props in class component is a bit different than functional components. Since we are inside a class, we must refer to the context where our props reside, again usually the component itself.

State

Whats state?

At it's most basic. State is a property of our class components. It is an object made of different key-value pairs of data that our application needs to keep track of.


                constructor() {
                    super();
                    this.state = {
                        username : '',
                        password: ''
                    }
                }

Heres an example of what a log in component's state could be. A user would need to enter a username and password to log into a website. This component's state is set up to track both as strings.

Using State


                    <h2>{ this.state.title }</h2>

There's no point in setting up a data store if you can't access anything in it. Since state is a property of our class components, and this gives us access to our current context, usually the class itself, we can easily access different values inside the state object.
